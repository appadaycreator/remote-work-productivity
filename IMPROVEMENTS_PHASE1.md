## 📋 改善提案（優先度順）

### P1. スコア表示・結果サマリーの改善

**目的**: ユーザーが自分の現状を「可視化」できるようにして、結果への納得度・シェア意欲を向上させる

**現状の問題**:
- 結果は「タイプ名 + 説明文」のみで、スコア（◎△×）の視覚的表示がない
- モバイルでは説明文が長すぎて読み疲れ
- どのカテゴリで弱いのか分からない（5問の回答結果が可視化されていない）

**仕様**:
1. 結果画面に「カテゴリ別スコア表示」を追加
   - 5つのカテゴリ（環境・集中力・切り替え・コミュ・自己管理）ごとに★3段階で表示
   - 回答内容から自動計算（選択肢インデックス = スコア）
   
2. 「総合スコア」を目立つ位置に表示
   - 0-15点の数値 + ゲージバー
   - 色分け：良好（青）→改善可能（黄）→要改善（赤）
   
3. モバイル時は説明文を「3行要約」に圧縮（詳細は折りたたみ可能）

**実装タスク**:
- [ ] `getResult()` 関数を拡張し、カテゴリ別スコア配列を返却
- [ ] 結果表示HTML内に `<div class="score-breakdown">` 要素を追加
- [ ] CSS: モバイル時は説明文の高さを制限 + "詳細を見る" 折りたたみ実装

**DoD**:
- [ ] 回答後、結果画面に総合スコア (0-15点) が表示される
- [ ] 5カテゴリのスコア★が正確に表示される
- [ ] モバイル (320px) で結果がスクロール不要に見える
- [ ] 「詳細を見る」で説明文が展開される（モバイルのみ）

### P2. SNS シェア機能の拡充 + スクリーンショット保存

**目的**: 診断結果をシェアしやすくして、バイラル効果 + SEO 向上

**現状の問題**:
- X (Twitter) と LINE のみシェア対応
- シェアテキストが固定で、自分のスコアが含まれていない
- スクリーンショット保存機能がない

**仕様**:
1. シェアテキストにスコアを動的に含める
   ```
   例：「テレワーク生産性診断で 総合スコア 12/15 (要改善型) を診断🏠
   あなたのタイプは？無料診断→ [URL]」
   ```

2. 「結果を画像で保存」ボタンを追加
   - HTML2Canvas または Canvas 使用
   - スコア + タイプ + 説明文を1枚の画像化
   - PNG ダウンロード

3. Facebook・LINE グループ共有対応（リンク共有も可能に）

**実装タスク**:
- [ ] `showResult()` 時にスコアを `window` に保存
- [ ] SNS シェアボタンに `encodeURIComponent()` で動的テキストを埋め込み
- [ ] HTML2Canvas ライブラリ読み込み + 「画像保存」ボタン実装
- [ ] 結果画面の CSS に `@media print` 対応（印刷でもスコア表示）

**DoD**:
- [ ] シェアテキストに「総合スコア XX/15」が含まれる
- [ ] 「結果を画像で保存」ボタンを押すと PNG がダウンロードされる
- [ ] 画像には総合スコア + タイプ + 簡潔な説明が含まれる
- [ ] Facebook シェアも対応（OGP 活用）

### P3. 「もう一度診断」フロー改善 + 以前の結果との比較

**目的**: リピート率向上 + ユーザー粘着性向上（次のアクション提示）

**現状の問題**:
- 「もう一度診断」ボタンを押すと `location.reload()` で全リセット
- 前回の結果が記録されないので、改善を追跡できない
- 結果を確認後、何をすればいいか不明確

**仕様**:
1. 結果確定時に localStorage に前回スコアを記録
   ```javascript
   localStorage.setItem('lastScore', totalScore)
   localStorage.setItem('lastResult', resultIndex)
   localStorage.setItem('lastDate', new Date())
   ```

2. 診断開始画面で「前回のスコア」を表示（あれば）
   ```
   「前回: 総合スコア 8/15 (環境整備型) — 2026年5月27日」
   ```

3. 結果画面に「次のアクション」セクションを追加
   - タイプ別の実行可能な TODO リスト（3-5個）
   - チェックボックス付き（localStorage に保存）

**実装タスク**:
- [ ] `showResult()` 時に localStorage 書き込み処理を追加
- [ ] 初期画面で localStorage から前回値を読み込んで表示
- [ ] 結果タイプごとに TODO リストを定義（データ化）
- [ ] CSS: TODO セクションのスタイル実装

**DoD**:
- [ ] 診断後、localStorage にスコア・結果・日時が保存される
- [ ] 2回目以降は「前回: XX/15」が表示される
- [ ] 結果画面に実行可能な TODO が 3-5 個表示される
- [ ] TODO チェック状態が localStorage に保存される

### P4. アクセシビリティ向上 + モバイルUX改善

**目的**: WCAG 2.1 AA レベル達成 + モバイルユーザーの離脱率低下

**現状の問題**:
- クイズオプションボタンのクリック判定が小さい（タッチ対応不十分）
- キーボードナビゲーション非対応（Tab キーで遷移不可）
- スマホで横向きの際、レイアウトが崩れる

**仕様**:
1. クイズオプションボタンの最小高さを 48px 統一（モバイルタッチ推奨）
2. キーボード操作対応
   - ↓/↑ で選択肢移動
   - Enter で選択確定
3. 画面回転対応（landscape モード）

**実装タスク**:
- [ ] `.quiz-opt` の `min-height: 48px` 確認
- [ ] `addEventListener('keydown')` でキーボードハンドリング実装
- [ ] `@media (orientation: landscape)` CSS 追加

**DoD**:
- [ ] モバイル (iPhone SE サイズ 375x667) で全機能が操作可能
- [ ] Tab キーで全選択肢を順巡できる
- [ ] Enter で選択肢が選べる
- [ ] 横向き（landscape）でレイアウト崩れなし
